## Use cases

- **Gate your CI on AI review** вЂ” one pipeline step posts a structured review
  to the MR/PR and exits **1** on a critical finding: the review *is* the
  quality gate. [Run in CI в†’](/ci/)
- **Review locally before you push** вЂ” `junior run` reviews your branch's diff
  right in the terminal, so the critical finding surfaces before the MR even
  exists. [Getting started в†’](/getting_started/)
- **Automate the routine** вЂ” a standup summary from the git log, triage of a
  log file, a drafted report: any prompt + shell command becomes a runbook.
  [Use cases в†’](/use_cases/)
- **Pipelines of juniors** вЂ” `junior run | junior run`: one junior does the
  task, the next reviews it and publishes only what passes.
  [Runbooks in YAML в†’](/script_runbooks/)

## How it works

Junior runs **runbooks**. A runbook is one task: collect в†’ one LLM call в†’ publish. Code review is the built-in flagship; the framework assumes nothing about code.

```
Collect (Runbook part)     в†’   Harness (the one AI step)   в†’   Publish (Runbook part)
-----------------------          -------------------------        -----------------------
anything code can gather:        one LLM call вЂ” the answer        anywhere code can deliver:
a git diff, an API response,     must match the runbook's         stdout / a file / PR or MR
a command's stdout, stdin        output schema                    comments / any command
```

**Runbook and Harness are independent** вЂ” any harness serves any runbook, because the output schema is a parameter, not a hard-coded type. The whole story: [Architecture](/architecture/) В· [Philosophy](/philosophy/).

## Install

```bash
uv tool install "junior @ git+https://github.com/mishachepi/junior.git"            # core
uv tool install "junior[codex,gitlab] @ git+https://github.com/mishachepi/junior.git"  # extras compose
```

The core is lean вЂ” CLI + the default `claudecode` harness, no LLM SDKs. Add an extra per harness/platform you actually use:

| Harness / platform | Install | Also requires |
|--------------------|---------|---------------|
| `claudecode` (default) | *(core)* | `claude` CLI installed and authenticated |
| `codex` | `junior[codex]` | `codex` CLI installed and authenticated |
| `pydantic` | `junior[pydantic]` | `OPENAI_API_KEY` or `ANTHROPIC_API_KEY` |
| `deepagents` вљ пёЏ deprecated | `junior[deepagents]` | `OPENAI_API_KEY` or `ANTHROPIC_API_KEY` вЂ” prefer `pydantic` |
| `pi` | *(core)* | `pi` CLI (`npm install -g @earendil-works/pi-coding-agent`); provider key вЂ” or none for local models |
| GitLab | `junior[gitlab]` | `GITLAB_TOKEN` |
| GitHub | `junior[github]` | `GITHUB_TOKEN` |
| Bitbucket DC | `junior[bitbucket]` | `BITBUCKET_*` env vars |

## Getting started

```bash
junior init                              # one-time setup wizard (picks your default runbook)
junior dry-run --runbook local_review    # free preview: exactly what the AI would get (no AI call)
junior run --runbook local_review        # review your local diff в†’ structured JSON verdict
junior run --runbook github_pr_review --publish   # review the PR + post comments
```

The guided five-minute walkthrough is [Getting Started](/getting_started/); every flag is in the [CLI reference](/cli/), CI recipes in [Run in CI](/ci/).
